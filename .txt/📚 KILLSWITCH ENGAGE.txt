📚 ANOTAÇÃO COMPLETA DO PROJETO KILLSWITCH ENGAGE
🎮 VISÃO GERAL
Projeto: Killswitch Engage – Sistema Inteligente de Recomendação de Jogos
Objetivo: Demonstrar habilidades em BD, Backend, ML/IA e Portfolio

🏗️ PIPELINE COMPLETA (6 FASES)

- **FASE 0: PLANEJAMENTO**: Definição do problema de negócio, requisitos e arquitetura do sistema.
- **FASE 1: FUNDAÇÃO (BD)**: Modelagem e criação do banco de dados PostgreSQL com 122k jogos.
- **FASE 2: EXPLORAÇÃO (EDA)**: Análise profunda de dados para extrair insights e preparar features.
- **FASE 3: BACKEND MÍNIMO**: Desenvolvimento da API robusta com FastAPI para servir o modelo.
- **FASE 4: MODELAGEM (IA/Recomendação)**: Treinamento e ajuste dos modelos de ML (RF, KMeans, LightFM, cGAN).
- **FASE 5: EXPERIMENTAÇÃO**: Otimização de hiperparâmetros e versionamento com MLflow e Optuna.
- **FASE 6: APRESENTAÇÃO (PORTFOLIO)**: Documentação final, entrega e visualização de resultados.

Modelo Lógico (Tabelas Detalhadas)
sql
1. usuarios (
    id, nome, email, data_cadastro, data_nascimento,
    pais, tipo_assinatura, ultimo_login, total_horas
)

2. jogos (
    id, titulo, descricao, data_lancamento, preco_base,
    desenvolvedor_id, avaliacao_media, total_avaliacoes
)

3. categorias (
    id, nome, descricao
) -- RPG, FPS, Estratégia, Aventura...

4. jogos_categorias (
    jogo_id, categoria_id
)

5. desenvolvedores (
    id, nome, pais, data_fundacao
)

6. biblioteca (
    usuario_id, jogo_id, data_aquisicao,
    tipo_aquisicao (compra/presente/promocao), preco_pago
)

7. sessoes_jogo (
    id, usuario_id, jogo_id, inicio, fim,
    horas_jogadas, plataforma (PC/console)
)

8. avaliacoes (
    usuario_id, jogo_id, nota (1-5), comentario,
    data, votos_uteis, recomendado (sim/não)
)

9. conquistas (
    id, jogo_id, nome, descricao, pontos,
    percentual_jogadores, tipo (secreta?)
)

10. conquistas_usuario (
    usuario_id, conquista_id, data_desbloqueio
)

11. precos_historicos (
    jogo_id, preco, data_inicio, data_fim,
    promocao_id, desconto_percentual
)

12. promocoes (
    id, nome, data_inicio, data_fim, tipo
)
Constraints e Índices
PKs em todas tabelas

FKs com ON DELETE CASCADE/RESTRICT

Índices compostos: (usuario_id, jogo_id) em sessoes

Índices: data_inicio, data_fim em precos

CHECK: nota between 1-5, horas_jogadas >= 0

UNIQUE: (usuario_id, jogo_id) na biblioteca

🧠 FASE 4: MODELAGEM (IA/Recomendação)
Subfases
text
4.1 BASELINE (regras simples)
    ├── Mais populares da semana
    ├── Mesma categoria do último jogo
    └── Lançamentos recentes

4.2 FILTRAGEM COLABORATIVA
    ├── User-based KNN
    ├── Item-based KNN
    └── Matrix Factorization (SVD)

4.3 BASEADO EM CONTEÚDO
    ├── Perfil do usuário (categorias)
    ├── TF-IDF em descrições
    └── Similaridade de jogos

4.4 MODELOS HÍBRIDOS
    ├── Weighted ensemble
    ├── Stacking
    └── LightFM (meta-features)

4.5 DEEP LEARNING (OPCIONAL)
    ├── Embeddings de jogos (Word2Vec)
    ├── Neural Collaborative Filtering
    └── LSTMs para padrões sequenciais

Tratamento de Problemas
Cold start usuário: popularidade + onboarding implícito

Cold start jogo: metadata + similaridade com jogos existentes

Sparsidade: matrix factorization + fallback rules

Scalability: índices + cache + aproximações
